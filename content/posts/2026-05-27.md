现在可以根据元素的容器应用条件 CSS，使用熟悉的语法：


```javascript
@container (min-width: 40rem) {
.some-elem {
font-size: 1.5rem;
}
}
```


#  **基本理念**


在过去几十年里，我们进行响应式设计的主要工具一直是媒体查询。最常见的是，我们使用视口宽度来有条件地应用一些 CSS：


```javascript
@media (min-width: 40rem) {
  .mobile-only {
    display: none;
  }
}
```


媒体查询很棒，但它们只关注全局属性，比如视口尺寸或操作系统的颜色主题。有时，我们希望根据本地属性有条件地应用 CSS，比如元素的容器大小。


例如，假设我们有一个 `ProfileCard` 组件，用于显示关于用户个人资料的关键信息：


![%E6%88%AA%E5%B1%8F2024-11-10_20.47.36.png](images/blog/2026-05-27/img-587b580b43.png)


在这种情况下，每个 `ProfileCard` 都很窄，因此信息垂直堆叠在 1 根高柱中。


在其他情况下，我们可能会有更多的空间。如果我们的 ProfileCard 能够根据可用空间自动切换布局，那岂不是很酷？


可能像这样：


![%E6%88%AA%E5%B1%8F2024-11-10_20.52.23.png](images/blog/2026-05-27/img-d7517d3f9d.png)


在某些情况下，我们可以使用媒体查询来实现这一点，如果我们的 `ProfileCard` 与视口大小成比例……但这并不总是如此。


例如，我们可能像这样将这些卡片排列在一个弹性网格中：


![%E6%88%AA%E5%B1%8F2024-11-10_20.53.09.png](images/blog/2026-05-27/img-e77d34cad2.png)


使用这种动态布局，每个 `ProfileCard` 将使用在可用空间量下最有意义的布局。这与视口大小无关！


显然，媒体查询不是这项工作的正确工具。相反，我们可以使用容器查询来解决这个问题。它看起来是这样的：


```javascript
.child-wrapper {
  container-type: inline-size;
}
.child {
  /* Narrow layout stuff here */
  @container (min-width: 15rem) {
    /* Wide layout stuff here */
  }
}
```


使用原生的 CSS 嵌套来将 `@container` 规则放在 `.child` 块内部，这样这个元素的 CSS 声明都位于同一块 CSS 中。


但是等等， `.child-wrapper` 是什么情况？那个 `container-type` 属性在做什么呢？


为了使用容器查询，我们首先需要明确定义其容器。这可能会产生一些意想不到的后果。


考虑以下场景：


```plain text
<style>
  p {
    width: fit-content;
    outline-style: solid;
  }
</style>
<p>
  <strong>Hello World!</strong>
</p>
```


`fit-content` 关键字，它是一个根据元素内容增长/缩小的动态值。


现在，假设我们想要根据其容器的大小增加那段粗体文本的 `font-size` 。我们可以想象这样做：


```javascript
p {
  width: fit-content;
  @container (max-width: 10rem) {
    strong {
      font-size: 3rem;
    }
  }
}
```


这似乎很有道理……当我们的父级 `<p>` 标签为 10rem 或更小时，我们将 `font-size: 3rem` 应用到内部的 `<strong>` 标签上。


当我们的容器为 10rem 或更小时，我们应用使容器超过 10rem 的样式。我们条件性地应用的 CSS 导致条件不再满足！但是，让我们真正地思考一下。当我们更改一个元素的 `font-size` 时，它不仅会影响字符的高度，还会影响元素的宽度


当我们的容器为 10rem 或更小时，我们应用使容器超过 10rem 的样式。我们条件性地应用的 CSS 导致条件不再满足！


当我们的 `<section>` 宽度小于 10rem 时，我们的条件就满足了，因此我们应用一些 CSS，增加字体大小。但这会导致我们的 `<p>` 标签扩展，从而使父 `<section>` 超出 `10rem` 阈值！在容器查询内部编写的 CSS 可能会影响容器本身，导致这些闪烁的 UI 无限循环。


# **The Containment API** 


通过在父元素上设置 `contain: size` ，我们切断这个连接。因此，容器的高度不再依赖于其内容。如果我们不指定显式的 `height` ，容器将折叠到 0px（加上填充）。


Containment API 的设计考虑了性能优化。CSS 是一种非常动态的语言，这意味着当事情发生变化时，浏览器通常需要做很多工作。例如：当我们调整蝾螈图片的高度时，这不仅会影响该演示中的元素，还会影响本文中随后的所有内容。像这样的段落会在每次尺寸变化时上下移动，导致布局重新计算和重绘。


因此，如果我们知道一个元素是自包含的并且不会影响其他任何东西，我们可以使用 `contain` 属性来让浏览器知道它可以跳过某些计算。对于 React 开发者来说，这是一个有用的类比：它有点像 `React.memo()` 。我们可以使用 `contain` 来选择退出我们知道是不必要的重新计算。


# **第一个容器查询**


考虑到所有这些背景信息，让我们编写一个“hello world”容器查询：


```javascript
<style>
  section {
    container-type: size;
  }

  @container (max-width: 12rem) {
    p {
      font-weight: bold;
      color: red;
    }
  }
</style>

<section>
  <p>
    This text becomes bold and red in
    small containers.
  </p>
</section>
```


首先，我们声明 `<section>` 元素是一个容器。这将允许其任何后代将其用作衡量标准，在满足某些条件时应用 CSS。


接下来，我们创建一个容器查询，选择容器内的 `<p>` ，并在容器宽度为 12rem 或更小时调整其外观样式。当满足此条件时，该块内的 CSS 将被应用，文本将变为粗体和红色。


如果您在具有大屏幕的设备上查看此内容，您可以亲自看到：通过点击并拖动分隔符或聚焦并使用左右箭头键来调整结果窗格的大小。


这个问题在于这个实现，当我们给我们的容器添加一些外观样式时，它就变得明显了。


```javascript
<style>
  section {
    container-type: size;
    background-color: peachpuff;
    border: 2px solid;
    padding: 8px;
  }
</style>

<section>
  <p>
    Uh oh! Something funky is happening here. For some reason, this paragraph of text is overflowing.
  </p>
</section>
```


![%E6%88%AA%E5%B1%8F2024-11-10_21.01.35.png](images/blog/2026-05-27/img-ce0f05474f.png)


像我们在 axolotl 例子中看到的那样，父元素不再动态响应用户。它不是根据段落内容增长以适应，而是塌陷成无；我们之所以能看到背景颜色，仅仅是因为这个元素恰好有一些填充！


当我们设置 `container-type: size` 时，我们告诉浏览器这个元素的布局不依赖于其子元素。这防止了我们之前看到的无限循环，但它也打破了我们关于 CSS 工作方式的一个核心假设！


我们通常不会去想这个问题，但在网页上，宽度和高度之间存在一个基本区别：

- 当涉及到宽度时，元素倾向于扩展，填充父元素提供的空间。
- 当涉及到高度时，元素往往会收缩包裹其子元素。

考虑一个没有任何 CSS 应用到的空 `<div>` 。它的高度将为 0px，但宽度不会是 0px。无论是否有内容，它都会扩展以填充整个水平空间。


当我们设置 `container-type: size` 时，我们告诉 CSS 忽略其内容，这意味着它将恢复到默认行为，即折叠到零！


幸运的是，我们可以为 `container-type` 属性使用另一个值， `inline-size` ：


```javascript
<style>
  section {
    container-type: inline-size;
    background-color: peachpuff;
    border: 2px solid;
  }

  @container (max-width: 12rem) {
    p {
      font-weight: bold;
      color: red;
    }
  }
</style>

<section>
  <p>
    Yay, it works! The parent’s height will change dynamically to accommodate its content, and the container query still fires when the container is resized!
  </p>
</section>
```


![%E6%88%AA%E5%B1%8F2024-11-10_21.02.48.png](images/blog/2026-05-27/img-01c61ca3b8.png)


该术语 `inline-size` 此处指代内联尺寸，通常是宽度。


本质上我们在这里说的是，元素的宽度不依赖于其内容。因此，它可以作为其子元素的一个测量标准。相比之下，元素的高度保留其基于内容的增长/缩小的默认行为。
