CSS Grid 是一种**布局系统**，将网页划分成一个个网格，可以任意组合不同的网格，做出各种各样的布局，可让轻松创建复杂且响应迅速的基于网格的布局。


它提供**基于二维网格**的布局系统，使其在处理行和列方面特别强大。


## 基础概念


**1.容器和项目**


采用网格布局的区域，称为"**容器**"（container）。容器内部采用网格定位的子元素，称为"**项目**"（item）。


**2.行和列**


容器里面的水平区域称为"行"（row），垂直区域称为"列"（column）。


![1_bg2019032502.png](images/blog/2024-09-15-css/img-3776a7b9c8.png)


上图中，水平的深色区域就是"行"，垂直的深色区域就是"列"。


**3.单元格**


行和列的交叉区域，称为"单元格"（cell）。


**4.网格线**


划分网格的线，称为"网格线"（grid line）。水平网格线划分出行，垂直网格线划分出列。


正常情况下，n行有n + 1根水平网格线，m列有m + 1根垂直网格线，比如三行就有四根水平网格线。


![1_bg2019032503.png](images/blog/2024-09-15-css/img-c15947f733.png)


## 实践


### 对网格轨道尺寸使用灵活的单位


不要为网格轨道尺寸指定固定像素值，而要使用灵活的单位，如_**fr**_或_**minmax**_。这是 CSS 网格最佳实践的核心之一，可使布局更适应不同的屏幕尺寸。


当使用固定像素值指定网格轨道尺寸时，实际上是在为布局设置刚性结构。随着屏幕尺寸的变化，固定像素值可能会导致布局扭曲，从而导致元素重叠或过度拉伸。


为了克服这些限制，CSS Grid 提供了灵活的单位，如_fr_和_minmax_，使布局能够优雅地适应不同的屏幕尺寸。


这些单元根据可用空间动态调整其尺寸，确保您的布局在各种设备上保持一致且具有视觉吸引力。


### 使用适当的网格结构


CSS Grid 最佳实践的一个重要方面是使用适当的网格结构。使用 CSS Grid 时，一个基本决定是选择显式网格还是隐式网格来定义网格容器内的网格项布局。

- 使用**显式网格**，可以精确控制内容的结构，定义了网格的行和列的数量和大小。
    - grid-template-columns: 定义列的尺寸和数量。
    - grid-template-rows: 定义行的尺寸和数量。
- 使用**隐式网格**时，当网格容器中的网格项超出了显式定义的网格区域时，自动创建的额外网格区域。这种方式不需要显式定义所有行和列，而是根据网格项的放置自动扩展。
    - grid-auto-columns 和 grid-auto-rows: 定义自动创建的列和行的尺寸。
    - grid-auto-flow: 控制网格项的放置顺序，通常是 row 或 column。
> 显式网格
> - **适用场景**：适用于需要精确控制布局的场景，例如固定布局、复杂的网页设计、需要明确定义区域的布局。
> - **优点**：精确控制、可读性高、调试方便、适合响应式设计。
> - **缺点**：灵活性较低、代码量大。
>
> 隐式网格
>
> - **适用场景**：适用于内容动态变化的场景，例如内容管理系统、数据驱动的布局、未知数量的项目。
> - **优点**：灵活性高、代码量少、适应性强。
> - **缺点**：控制较弱、调试困难、布局不明确。
>

### 显式网格


显式网格可以精确控制布局结构，并提高设计的可预测性。它对于包含固定项目的组件（例如导航栏或卡片）尤其有用。


eg：


使用了显式的网格布局来创建定价卡组件，可以适应不同的屏幕尺寸和设备。


_**auto-fit**_ **和** _**minmax**_ 的组合确保布局无缝适应不同的屏幕尺寸和设备。


```javascript
.pricing-container {
 	display: grid;
 	grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
 	gap: 25px;
 	justify-content: center;
 	margin: 50px;
}
```


### 隐式网格


隐式网格会根据具体的内容自动调整，当没有足够的空间或者显式的网格轨道来设置网格项目，此时网格项目就会自动创建隐式网格。


eg：有一个产品页面，允许用户通过点击“添加产品”按钮动态添加产品。每个产品都有名称和价格。


在这里，由于元素是动态生成的，并且内容的数量可能会有所不同，因此使用隐式网格与显式网格结合来定义布局。


1.使用grid-template-columns属性明确定义列：


```javascript
.product-list {
 grid-template-columns: repeat(
   auto-fill,
   minmax(200px, 1fr)
 ); 
}
```


使用auto-fill和minmax(200px, 1fr)可以创建容器中可以容纳的尽可能多的列，每个列宽至少为200px，但如果有额外空间，则可以增长 1fr。


2.使用隐式网格来设置行的大小，因为内容可能有不同的高度：


```javascript
grid-auto-rows: minmax(
   250px,
   auto
 );
```


这确保隐式行具有最小高度250px，但如果内容需要更多空间，则它们可以根据内容的高度自动动态增加。


.product-list 网格容器中结合了显式网格（用于列）和隐式网格（用于具有动态大小的行）—— 允许拥有结构化的列布局，同时根据内容灵活地调整行高。


### 选择定位方法


使用网格布局时会出现一个问题，即是否使用**grid-template-area**或**grid-template-columns/rows**来定位元素 —— 根据布局的复杂性和性质来使用它们。

> grid-template-areas
>
> **优点**:
>
> 1. **直观性**：通过命名网格区域，可以非常直观地定义布局结构。
> 2. **易读性**：布局的命名使得代码更易于理解和维护。
> 3. **灵活性**：可以轻松定义复杂的布局结构，如多行多列的组合。
>
> **缺点**:
>
> 1. **重复性**：如果网格布局中的多个区域使用相同的内容，需要重复多次定义。
> 2. **限制性**：对于动态内容或未知数量的项目，使用 grid-template-areas 可能不够灵活。
>
> **应用场景**：
>
> - 当布局结构固定且可以通过命名区域清晰描述时。
> - 对于具有明确区域划分的设计，如导航菜单、页眉页脚等。
>
> **使用 grid-template-columns/grid-template-rows**
>
>
> **优点**:
>
> 1. **灵活性**：可以轻松适应不同数量的内容或动态变化的内容。
> 2. **简洁性**：对于简单的列和行布局，代码更加简洁。
> 3. **控制性**：可以精确控制列和行的大小，包括固定尺寸、比例、最小最大值等。
>
> **缺点**:
>
> 1. **复杂性**：对于复杂的布局，需要更多的代码来定位和调整元素。
> 2. **直观性**：对于初学者，不使用命名区域可能使得布局结构不够直观。
>
> **应用场景**：
>
> - 当需要创建响应式布局，能够适应不同屏幕尺寸和内容变化时。
> - 对于简单的列和行布局，如卡片列表、分栏内容等。
>

### **grid-template-areas**


**grid-template-areas** CSS 属性是网格区域 grid areas 在 CSS 中的特定命名。


如果希望布局简单易懂，特别是当具有特定用途的不同部分时 —— 推荐使用_grid-template-areas_


eg：有一个布局，其中包含标题、导航栏、主部分、侧边栏和页脚。


```javascript
.container {
 display: grid;
 grid-template-areas:
   "header header header"
   "nav main sidebar"
   "footer footer footer";
 min-height: 100vh;
}
```


已经命名了网格区域 —— 将元素分配给这些区域 —— 要将内容放置在网格区域中，使用grid-area属性：


```javascript
header {
  grid-area: header;
}

nav {
  grid-area: nav;
}
```


这会指示“特定元素”内的内容进入网格中名为“该元素”的区域。


### grid-template-columns/rows


如果更注重精确控制并保持简单，尤其是涉及每个部分的大小和位置 —— 使用grid-template-columns或grid-template-rows属性。


eg：


```javascript
.container {
 display: grid;
 grid-template-columns: 1fr 2fr;
 align-items: center;
 padding: 0 60px;
 gap: 20px;
 min-height: 100vh;
}
```


### 使用 Flexbox 网格


将网格与 Flexbox 相结合是推荐的 CSS 网格最佳实践之一。CSS 网格对于创建灵活且响应迅速的布局非常有用，但它并不总是每种情况下的最佳解决方案。在某些情况下，将网格布局与其他布局机制（例如 Flexbox 或浮动）相结合可能会更有效。


一种常见的做法是将 Grid 与 CSS Flexbox 结合使用，例如在卡片布局中。

- CSS Grid 设置布局的主要结构，定义整体网格和主要元素的定位。
- 在每个网格单元内使用 Flexbox 来管理项目的内部排列。

尽管 CSS Grid 适合设置行和列，但 Flexbox 适合在行或列中整齐地排列元素，尤其是当它们大小不同时。


通过采用这两种方法，可以使用 CSS Grid 提供整体框架，同时依靠 Flexbox 处理布局中每个部分的细节。这种组合允许在高级结构和精细控制之间取得平衡，从而以简洁高效的方式组织内容。


eg:使用 CSS Grid 和 Flexbox 创建了一个基于响应式卡片的布局。


.grid -container类定义网格布局，它开始是单列布局，在宽度超过500px的屏幕上切换到两列布局，在宽度超过850px的屏幕上切换到四列布局。


```javascript
.grid-container {
  width: 90%;
  max-width: 1240px;
  margin: 0 auto;
  display: grid;
  grid-template-columns: 1fr;
  grid-template-rows: auto;
  grid-gap: 20px;
}
```


Grid 布局与 Flex 布局 有一定的相似性，都可以指定容器内部多个项目的位置。但是，它们也存在区别。

- Flex 布局是轴线布局，只能指定"项目"针对轴线的位置，可以看作是**一维布局**。
- Grid 布局则是将容器划分成"行"和"列"，产生单元格，然后指定"项目所在"的单元格，可以看作是**二维布局**。
