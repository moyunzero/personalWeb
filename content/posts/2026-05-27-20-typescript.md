# **1. 非空类型（** **`NonNullable`** **）**


TypeScript 提供了 `NonNullable` 工具来从类型中消除 `null` 和 `undefined` ，这可以帮助您避免意外的空值


```typescript
type User = { name: string; age?: number | null };
const user: NonNullable<User["age"]> = 30; // ✅ No null or undefined allowed
```


# **2. 使用** **`Partial`** **以获得灵活性**


`Partial<T>` 工具使类型中的所有属性都变为可选，当你只更新对象字段的子集时，这非常棒。


```typescript
interface User {
  name: string;
  age: number;
  email: string;
}

const updateUser = (user: Partial<User>) => {
  // You can pass only the fields you want to update
  return { ...user, updatedAt: new Date() };
};

updateUser({ name: 'John' }); // No need to provide the entire object
```


# **3. 利用** **`Readonly`** **进行不可变数据** 


当你需要在 TypeScript 中实现不可变性时， `Readonly<T>` 使类型的所有属性不可变，防止重新赋值。


```typescript
const config: Readonly<{ apiUrl: string; retries: number }> = {
  apiUrl: 'https://api.example.com',
  retries: 5
};

config.apiUrl = 'https://newapi.com'; // ❌ Error: Cannot assign to 'apiUrl' because it is a read-only property
```


# **4. 动态属性类型映射**


映射类型允许您通过转换现有类型来创建新类型。这对于创建对象类型的变体很有用。


```typescript
type Status = 'loading' | 'success' | 'error';
type ApiResponse<T> = {
  [K in Status]: T;
};

const response: ApiResponse<string> = {
  loading: 'Fetching...',
  success: 'Data loaded',
  error: 'Something went wrong'
};
```


# **5. 带有可选元素的元组类型**


你知道 TypeScript 允许元组中有可选元素吗？当处理可变函数参数时，这非常棒。


```typescript
type UserTuple = [string, number?, boolean?];

const user1: UserTuple = ['Alice'];          // ✅ Just the name
const user2: UserTuple = ['Bob', 30];        // ✅ Name and age
const user3: UserTuple = ['Charlie', 25, true]; // ✅ Full tuple
```


# **6. 使用穷尽检查的联合类型** 


确保您使用联合类型和详尽的检查在 switch 语句中处理所有可能的用例。


```typescript
type Status = 'open' | 'closed' | 'pending';

function handleStatus(status: Status) {
  switch (status) {
    case 'open':
      return 'Opened';
    case 'closed':
      return 'Closed';
    case 'pending':
      return 'Pending';
    default:
      const exhaustiveCheck: never = status; // ❌ Error if a new status type is added but not handled
      return exhaustiveCheck;
  }
}
```


# **7. 实用类型** **`Omit`** **用于排除键**


有时你需要创建一个排除某些键的对象类型。 `Omit` 在这里是你的好朋友！


```typescript
interface Todo {
  title: string;
  description: string;
  completed: boolean;
}

type TodoPreview = Omit<Todo, 'description'>;

const todo: TodoPreview = {
  title: 'Learn TypeScript',
  completed: false
};
```


# **8. 使用** **`in`** **和** **`instanceof`** **进行类型缩小**


```typescript
function processInput(input: string | number | { title: string }) {
  if (typeof input === 'string') {
    return input.toUpperCase(); // Narrowed to string
  } else if (typeof input === 'number') {
    return input * 2; // Narrowed to number
  } else if ('title' in input) {
    return input.title; // Narrowed to object with title property
  }
}
```


# **9. 高级类型逻辑的条件类型** 


条件类型为您提供了基于条件转换类型的惊人灵活性。


```typescript
type IsString<T> = T extends string ? true : false;

type CheckString = IsString<'Hello'>; // true
type CheckNumber = IsString<42>; // false
```


# **10. 使用** **`as const`** **表示不可变字面量类型**


`as const` 非常适合冻结值并确保 TypeScript 将它们视为字面量类型，而不是可变值。


```typescript
const COLORS = ['red', 'green', 'blue'] as const;

type Color = typeof COLORS[number]; // 'red' | 'green' | 'blue'
```


# **11. 提取并排除以精炼类型**


使用 `Extract` 和 `Exclude` 来过滤或从联合中挑选特定类型。


```typescript
type T = 'a' | 'b' | 'c';
type OnlyAOrB = Extract<T, 'a' | 'b'>; // 'a' | 'b'
type ExcludeC = Exclude<T, 'c'>; // 'a' | 'b'
```


# **12. 自定义验证的类型守卫**


创建您自己的类型守卫以在运行时动态地细化类型。


```typescript
function isString(input: any): input is string {
  return typeof input === 'string';
}

const value: any = 'Hello';

if (isString(value)) {
  console.log(value.toUpperCase()); // Safe: value is a string here
}
```


# **13. 使用** **`Record`** **表示动态对象类型**


当你需要一个具有动态键的对象类型时， `Record<K, V>` 是完美的选择。


```typescript
type Role = 'admin' | 'user' | 'guest';
const permissions: Record<Role, string[]> = {
  admin: ['read', 'write', 'delete'],
  user: ['read', 'write'],
  guest: ['read']
};
```


# **14. 使用索引签名的动态类属性**


索引签名允许您创建具有动态命名属性的对象或类。


```typescript
class DynamicObject {
  [key: string]: any;
}

const obj = new DynamicObject();
obj.name = 'Alice';
obj.age = 30;
```


# **15.** **`never`** **不可能状态类型**


`never` 类型表示永远不会发生的情况。它通常用于穷举检查。


```typescript
function assertNever(value: never): never {
  throw new Error(`Unexpected value: ${value}`);
}
```


# **16. 可选链式调用，安全属性访问** 


使用可选链（ `?.` ）安全地访问深层嵌套属性，无需担心 `undefined` 错误。


```typescript
const user = { profile: { name: 'John' } };
const userName = user?.profile?.name; // 'John'
const age = user?.profile?.age ?? 'Not provided'; // Fallback to default
```


# **17. 默认值与空合并（** **`??`** **）**


使用空值合并运算符仅在原始值为 `null` 或 `undefined` 时提供回退值。


```typescript
const input: string | null = null;
const defaultValue = input ?? 'Default'; // 'Default'
```


# **18. 使用** **`ReturnType`** **推断返回类型**


该 `ReturnType<T>` 工具提取函数的返回类型，当处理复杂类型时非常有用。


```typescript
function getUser() {
  return { name: 'John', age: 30 };
}

type UserReturn = ReturnType<typeof getUser>; // { name: string; age: number; }
```


# **19. 函数中的类型参数** 


泛型类型参数使您的函数在不同类型之间灵活且可重用。


```typescript
function identity<T>(value: T): T {
  return value;
}

identity<string>('Hello'); // 'Hello'
identity<number>(42); // 42
```


# **20. 结构组合的交集类型** 


```typescript
type Admin = { privileges: string[] };
type User = { name: string };

type AdminUser = Admin & User;

const adminUser: AdminUser = {
  privileges: ['admin', 'editor'],
  name: 'Alice'
};
```
