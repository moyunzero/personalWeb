### **1. 什么是 Git？**


Git 是一个分布式版本控制系统，它帮助开发者在任何规模的项目上进行协作。


### **2. 什么是分布式版本控制系统？**


分布式版本控制系统是一种帮助你跟踪项目文件更改历史的系统。这个更改历史存储在你的本地机器上，如果出现问题，你可以轻松地回退到项目的先前版本。


### **3. Git 的优势**

- **跟踪更改：** 记录代码的每次修改，知道谁改了什么。
- **团队协作：** 多人同时开发，合并代码。
- **版本管理：** 方便回滚到之前的版本，避免错误。

### **4. 常用 Git 命令**


### **4.1 初始化仓库**


```shell
git init
```

- 在当前目录创建一个新的 Git 仓库。

### **4.2 克隆仓库**


```shell
git clone [repository-url]
```

- 从远程仓库（如 GitHub）复制代码到本地。

### **4.3 查看状态**


```shell
git status
```

- 查看当前目录中哪些文件被修改、新增或删除。

### **4.4 添加文件到暂存区**


```shell
git add [file-name]
```

- 将文件添加到暂存区，准备提交。

### **4.5 添加所有文件到暂存区**


```shell
git add .
```

- 将项目中的所有文件添加到暂存区。

### **4.6 提交更改**


```shell
git commit -m "提交信息"
```

- 将暂存区的更改提交到本地仓库，并附上描述信息。

### **4.7 查看更改历史**


```shell
git log
```

- 查看所有提交记录。

### **4.8 远程仓库操作**


```shell
git remote add origin [repository-url]
```

- 添加远程仓库地址。

```shell
git push origin master
```

- 将本地更改推送到远程仓库。

```shell
git pull origin master
```

- 从远程仓库拉取最新代码。

### **4.9 创建分支**


```shell
git branch [branch-name]
```

- 创建一个新的分支。

### **4.10 切换分支**


```shell
git checkout [branch-name]
```

- 切换到指定分支。

### **4.11 合并分支**


```shell
git merge [branch-name]
```

- 将指定分支的更改合并到当前分支。

### **5. 常见问题解决**


### **5.1 如何撤销修改？**

- 如果文件还没有提交：

    ```shell
    git checkout -- [file-name]
    ```

- 如果文件已经提交：

    ```shell
    git reset HEAD~1
    ```


### **5.2 如何解决冲突？**

- 当合并分支时出现冲突，手动编辑冲突文件，解决冲突后：

    ```shell
    git add [file-name]
    git commit -m "解决冲突"
    ```


### **5.3 如何查看远程仓库信息？**


```shell
git remote -v
```


### **6. Git 工作流程**

1. **克隆仓库：** 从远程仓库获取代码。
2. **创建分支：** 在新分支上进行开发。
3. **添加和提交：** 将更改添加到暂存区并提交。
4. **合并分支：** 将分支合并到主分支。
5. **推送更改：** 将更改推送到远程仓库。

### **7. Git 的优点**

- **版本控制：** 轻松回滚到之前的版本。
- **团队协作：** 多人同时开发，合并代码。
- **分支管理：** 在不同分支上进行实验，不影响主项目。

### **8. 高级命令**


### **8.1 查看更改历史包括更改内容**


```shell
git log -p
```

- 查看提交历史，包括所有文件及其更改。

### **8.2 查看特定提交**


```shell
git show commit-id
```

- 查看特定的提交信息。

### **8.3 查看更改统计**


```shell
git log --stat
```

- 查看每个提交的更改统计，包括更改的行数和文件名。

### **8.4 查看更改使用 "diff"**


```shell
git diff
```

- 查看未暂存的更改。

```shell
git diff all_checks.py
```

- 查看特定文件的更改。

```shell
git diff --staged
```

- 查看已暂存的更改。

### **8.5 交互式添加更改**


```shell
git add -p
```

- 交互式地选择要暂存的更改。

### **8.6 删除跟踪的文件**


```shell
git rm filename
```

- 删除文件并记录删除原因。

### **8.7 重命名文件**


```shell
git mv oldfile newfile
```

- 重命名文件并记录更改。

### **8.8 忽略文件**

- 创建一个 `.gitignore` 文件并提交它。

### **8.9 回退未暂存的更改**


```shell
git checkout filename
```


### **8.10 回退已暂存的更改**


```shell
git reset HEAD filename
```

- 或者使用 `p` 选项指定要重置的更改。

### **8.11 修改最近的提交**


```shell
git commit --amend
```

- 修改最近的提交，添加或更改提交信息。

### **8.12 回退最近的提交**


```shell
git revert HEAD
```

- 创建一个新的提交，撤销最近的提交。

### **8.13 回退旧的提交**


```shell
git revert commit_id_here
```

- 回退指定的旧提交。

### **8.14 显示提交日志作为图表**


```shell
git log --graph --oneline
```

- 以图表形式显示提交日志。

### **8.15 显示所有分支的提交日志**


```shell
git log --graph --oneline --all
```


### **8.16 中止冲突的合并**


```shell
git merge --abort
```

- 中止当前的合并操作。

### **8.17 添加远程仓库**


```shell
git remote add origin https://repo_here
```

- 添加远程仓库地址。

### **8.18 查看远程仓库 URL**


```shell
git remote -v
```


### **8.19 查看远程仓库信息**


```shell
git remote show origin
```


### **8.20 推送更改到远程仓库**


```shell
git push
```


### **8.21 从远程仓库拉取更改**


```shell
git pull
```


### **8.22 查看远程分支**


```shell
git branch -r
```


### **8.23 从远程仓库获取更改**


```shell
git fetch
```


### **8.24 查看远程仓库的提交日志**


```shell
git log origin/main
```


### **8.25 合并远程仓库到本地**


```shell
git merge origin/main
```


### **8.26 更新远程仓库信息**


```shell
git remote update
```


### **8.27 推送新分支到远程仓库**


```shell
git push -u origin branch_name
```


### **8.28 删除远程分支**


```shell
git push --delete origin branch_name_here
```


### **8.29 使用 Git 变基**


```shell
git rebase branch_name_here
```

- 将一个分支上的工作转移到另一个分支上。

### **8.30 交互式变基**


```shell
git rebase -i master
```

- 交互式变基，可以修改提交历史。

### **8.31 强制推送**


```shell
git push -f
```

- 强制推送更改，谨慎使用。
