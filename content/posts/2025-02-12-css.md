# Scoping的定义


scoping选择提供了一种仅在DOM的某个区域内选择HTML元素的方法。scope可以帮助你针对HTML的特定部分。这让你对CSS样式有了更多的控制权。


@scope at-rule会开启一个作用域块，我们在括号中指定一个作用域根。这为指定根的子元素创建了一个作用域块。


**基本语法：**


```javascript
@scope (root) {
   element {
     style
   }
 }
```


我们在此范围内声明的任何 CSS 将仅适用于在该范围内选择的元素。


# 设定 CSS 的作用范围


作用域限制可以防止你的样式从一个组件泄露出去并影响其他组件。在某种程度上，作用域提供了一种“封装”CSS的方法。你的选择器将只应用于你想要的特定元素。


没有作用域，你需要为每个元素添加一个类，并且类名必须非常具体。这可能会导致样式和命名冲突，使大项目难以维护。


看一下下面的代码：


```javascript
p {
  color: red;
}
```


这个p选择器将适用于代码中的所有p元素。你可能希望对某些段落应用不同的样式。当然，你可以为每个需要不同样式的元素分配一个唯一的类名。然而，想出不同的类名可能是很累的。随着项目的扩大，这可能导致名称冲突和特异性问题。通常，我们使用特异性和出现顺序来处理这种例外情况，但在某些情况下，这会导致样式冲突。


范围定义消除了为每个元素添加类名的需求。你可以在范围块内使用简单的元素选择器。你不必担心命名规范或样式应用到你不想要的区域。


# @scope如何工作？


`<div class="tag">
  <p>Hello world</p>
</div>

<div class="card">
  <p>Hello everyone</p>
</div>`


在此 HTML 中，我们有两个类名为.tag和.card的div元素。两个div元素都嵌套有p 。


将此代码片段复制到您的 CSS 中：


`@scope (.tag) {
  p {
    background: red;
  }
}`


# 如何使用@scope


可以通过多种方式使用@scope规则来定位所需的某些元素。


## 针对单个元素


`<div class="jane">
  <p>my name is jane</p>
</div>`


在此代码中，我们有一个带有.jane类的div和嵌套在其中的一个段落。


`@scope (.jane) {
  p {
    color: grey;
  }
}`


上面的代码仅适用于范围根.jane内的p元素。


## 针对多个元素


你可以同时对两个根进行作用域处理。假设你有两个或更多的组件，希望对它们的嵌套元素应用相同的样式。你可以将它们放在一个作用域中，而不是为每个组件单独创建一个作用域。


`<div class="mike">
  <p>My name is Mike</p>
</div>

<div class="jane">
  <p>My name is Jane</p>
</div>`


此代码由两个div元素组成，类名分别为.mike和.jane 。每个div内部都嵌套有一个p元素。


下面的 CSS 代码将同时作用于.mike和.jane组件。


`@scope (.mike, .jane) {
  p {
    color: grey;
  }
}`


此样式将应用于.mike和.jane根中的p元素。


# 设置范围限制


@scope 允许你定义作用域的开始和结束。在层叠样式表中，你应用于父元素的每一个样式都会自动应用到该父元素的所有子元素。使用作用域，你可以在特定的点停止层叠。这被称为donut作用域。


要创建环形范围，您需要定义范围的上限和下限。这将在范围中创建一个洞，其中样式不适用。


`<div class="birthday-card">
  <h1>hello mary!</h1>
  <p>have a nice birthday</p>
  <div class="gift-card">
    <h1>here's your gift</h1>
    <p>Have a nice day</p>
  </div>
</div>`


在这里，我们有一个带有.birthday-card类的父div 。在父div内，有一个类为.gift-card子div 。


下面的 CSS 代码将只适用于范围上边界内的段落。范围下边界内的段落被视为超出范围。


`@scope (.birthday-card) to (.gift-card) {
  p {
    border: 2px solid red;
  }
}`


这里， .birthday-card div是范围的上边界， .gift-card div是范围的下边界（范围限制）。此范围从.birthday-card开始，到.gift-card结束。


# 解决 @scope 的特异性问题


在层叠样式表中，样式的应用是基于特异性和出现顺序的。如果两个具有相同特异性的声明都针对同一个元素，后出现的声明将覆盖之前所有的声明。这在大多数情况下可能会导致很多样式冲突。


`<div class="red-color">
  <a href="#">I am a red link</a>
</div>

<div class="yellow-color">
  <a href="#">I am a yellow link</a>
  <div class="red-color">
    <a href="#"> I am a red link</a>
  </div>
</div>`


上面的代码有一个.red-color div和一个.yellow-color div以及另一个.red-color div嵌套在其中。


`.red-color a {
  color: red;
  border: 3px solid red;
}

.yellow-color a {
  color: yellow;
  border: 3px solid yellow;
}`


如上所示，.yellow-color 类覆盖了 .red-color 类。链接的颜色是黄色而不是红色。这两个选择器具有相同的特异性，但 .yellow-color 类由于出现顺序在前而胜出。解决此问题的最佳方法是使用作用域接近性。


## 范围邻近性


范围允许我们根据范围根和目标元素之间的接近度来覆盖样式。在范围中，事物的顺序并不重要；重要的是接近度，而不是出现的顺序。


如果元素具有相同的特异性，那么作用域的接近程度将起作用。如果它们具有相同的作用域接近程度，那么将应用出现顺序。然而，特异性更高的元素仍然会覆盖接近程度。


当两个作用域声明针对同一元素时，级联将优先考虑作为目标元素最近父级的作用域根。


`@scope (.red-color) {
  a {
    color: red;
    border: 3px solid red;
  }
}

@scope (.yellow-color) {
  a {
    color: yellow;
    border: 3px solid yellow;
  }
}`


此处的样式将根据范围接近度来应用。 .red-color类是最接近红色链接的父级。这意味着.red-color类具有更强的范围邻近性，因此在这种情况下它获得更高的优先级。


看另一个例子：


`<div class="lightblue">
  <div class="lightpink">
    <button>CLICK ME</button>
  </div>
</div>`


在此 HTML 中，我们有一个类名为.lightblue的父div 。在这个父级中，我们有一个.lightpink div ，其中嵌套了一个button 。


`@scope (.lightblue) {
  button {
    background: lightblue;
  }
}

@scope (.lightpink) {
  button {
    background: lightpink;
  }
}`


.lightpink类将应用于此处，因为它是按钮最接近的父级。


# :scope 伪类


`:scope` 选择器定位作用域根元素。它与其他伪类具有相同的特异性，即 0,1,0。


`<div class="flex-container">
  <div class="flex-item">
    <p>Hello World</p>
  </div>
  <div class="flex-item">
    <p>Hello Universe</p>
  </div>
</div>`


此 HTML 有一个类名为.flex-container的父div 。它包含两个具有.flex-item类的子div 。


`@scope (.flex-container) {
  :scope {
    max-width:250px;
    display: flex;
    gap:20px;
    border: 3px solid red;
  }
  p {
    color: yellow;
  }
}`


:scope选择器将设置根元素的样式， p将设置根元素中段落的样式。


你也可以使用&来定位根。在嵌套中通常使用&，但它在作用域中同样有效。


`@scope (.flex-container) {
  & {
    max-width:250px;
    display: flex;
    gap:20px;
    border: 3px solid red;
  }
  p {
    color: yellow;
  }
}`


这将产生与使用:scope相同的结果。尽管:scope和&都以根为目标，但它们在选择器匹配和特异性方面有所不同。

- :scope` 选择器的优先级与其他伪类相同，而 `&` 的优先级会根据根元素而变化。
- :scope选择器仅匹配范围的根，而&匹配根的选择器。

# 使用@scope应用内联样式


你可以通过嵌入内联样式标签，将 @scope 直接添加到你的 HTML 代码中。使用内联样式时，你不需要为你的作用域指定根节点。该作用域将自动应用于样式标签的父级元素。


`<div class="inline-scope">
  <a href="#">I am a scoped link</a>
  <p>I am a scoped paragraph</p>
  <style>
    @scope {
      :scope {
        background: grey;
      }
      a:hover {
        color: red;
      }
      p {
        color: pink;
      }
    }
  </style>
</div>`


在这里，我们有一个div ，其中嵌套了一个内联style标签。此style标签包含将应用于此父div及其子元素的范围样式。


在上面的示例中，style标签嵌套在.inline-scope div内，该div作为作用域根。因此，作用域将仅适用于该div及其嵌套元素。


# 浏览器支持和限制


@scope 规则在所有主流浏览器中都得到了良好的支持，除了 Firefox。它可以在 Chrome、Safari 和 Edge 中正常工作。作用域正逐渐在开发者中获得人气，并且很快将在所有浏览器中获得完全支持。

# 小结


Scope 让你可以明确地决定代码中你想要进行样式设置的区域。作用域增强了开发者的体验，使编写代码变得更加容易。开发者不再需要担心写很长且过于具体的类名。不存在名称冲突、样式冲突或 CSS 样式溢出并影响其他元素的风险。
