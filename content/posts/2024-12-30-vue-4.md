# **1. 使用数据存储模式以避免属性钻取**


当应用增长时，我们经常发现自己需要将属性向下传递多层，只是为了将用户数据或共享状态传递到正确的组件中。因此，我们将全局状态放置在一个单独的可组合组件中。


任何组件都可以使用它。


这种模式将共享数据（如用户信息）移动到全局响应式存储中。而不是通过 props 和事件推送 `user` 数据，每个组件都直接从这个存储中读取。


它简化了我们的整个数据流。


```javascript
import { reactive, toRefs } from 'vue'

const state = reactive({
  user: {
    name: '',
    isLoggedIn: false,
  }
})

export function useUserStore() {
  const { user } = toRefs(state)

  function login(name) {
    user.value.name = name
    user.value.isLoggedIn = true
  }

  function logout() {
    user.value.name = ''
    user.value.isLoggedIn = false
  }

  return {
    user,
    login,
    logout
  }
}
```


这里我们定义了一个全局的 `state` 对象，持有 `user` 。 `useUserStore()` 函数返回引用和方法，使它们在任何地方都可以使用。


我们可以在主组件中使用它：


```javascript
<template>
  <div>
    <div v-if="user.isLoggedIn">
      Welcome, {{ user.name }}!
      <button @click="logout">Logout</button>
    </div>
    <div v-else>
      <input v-model="username" placeholder="Enter name" />
      <button @click="loginUser">Login</button>
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useUserStore } from './useUserStore'

const { user, login, logout } = useUserStore()
const username = ref('')

function loginUser() {
  if (username.value.trim()) {
    login(username.value)
  }
}
</script>
```


这使我们直接访问 `user` 。我们不再需要传递它。


现在全局状态已经建立，让我们转向下一个模式。


# **2. 使用轻量级可组合函数来分离逻辑**


我们经常在可组合组件中封装过多的逻辑。这使得它们更难进行测试。


如果我们把业务逻辑放入纯函数中，只使用可组合物进行响应式连接呢？


这是轻量级可组合函数的作用。


我们将在简单的 JS 文件中保留计算或格式化。组合式仅引用它。现在测试更简单了——你可以作为一个普通函数测试你的逻辑，无需 Vue。


```javascript
export function generateGreeting(name) {
  if (!name) return 'Hello, Guest!'
  return `Hello, ${name}!`
}
```


```javascript
import { computed } from 'vue'
import { generateGreeting } from './lib/userLogic'
import { useUserStore } from './useUserStore'

export function useGreeting() {
  const { user } = useUserStore()

  const greeting = computed(() => generateGreeting(user.value.name))

  return { greeting }
}
```


我们从 `userLogic.js` 导入 `generateGreeting()` 。组合函数 `useGreeting()` 仅设置一个计算值。在 `useGreeting()` 内部不需要复杂的逻辑。只是一个简单的桥梁。


我们现在可以在我们的 `App.vue` 组件中使用它：


```javascript
<template>
  <div>
    <div v-if="user.isLoggedIn">
      {{ greeting }}
      <button @click="logout">Logout</button>
    </div>
    <div v-else>
      <input v-model="username" placeholder="Enter name" />
      <button @click="loginUser">Login</button>
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useUserStore } from './useUserStore'
import { useGreeting } from './useGreeting'

const { user, login, logout } = useUserStore()
const { greeting } = useGreeting()
const username = ref('')

function loginUser() {
  if (username.value.trim()) {
    login(username.value)
  }
}
</script>
```


现在逻辑独立，测试变得简单。


# **3. 使用选项对象进行可组合配置**


随着我们的可组合组件增加功能，我们可能会添加更多参数。


但是传递多个参数可能会感觉脆弱。相反，我们可以传递一个带有命名选项的单个对象。这使得添加新设置变得容易。


我们将以功能标志为例。


我们从 API 加载标志或使用本地默认值。如果我们使用了位置参数，改变顺序或添加新参数可能会破坏现有的调用。使用选项对象，我们可以轻松扩展配置而不会破坏任何东西：


```javascript
import { ref, watchEffect } from 'vue'

export function useFeatureFlags(options = {}) {
  const {
    loadMethod = () => Promise.resolve({ darkMode: false }),
    defaultFlags = { darkMode: false }
  } = options

  const flags = ref({ ...defaultFlags })

  watchEffect(async () => {
    const loaded = await loadMethod()
    flags.value = { ...defaultFlags, ...loaded }
  })

  return { flags }
}
```


我们从 `options` 对象中解构默认值。


如果调用者没有传递 `loadMethod` ，我们将回退到简单的默认设置。如果我们想稍后添加另一个选项——比如 `cacheTime` ——我们只需在这里添加即可。


无需修改调用代码！


让我们将其添加到我们的 `App.vue` 组件中：


```javascript
<template>
  <div :class="{ dark: flags.darkMode }">
    <div v-if="user.isLoggedIn">
      {{ greeting }}
      <button @click="logout">Logout</button>
    </div>
    <div v-else>
      <input v-model="username" placeholder="Enter name" />
      <button @click="loginUser">Login</button>
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useUserStore } from './useUserStore'
import { useGreeting } from './useGreeting'
import { useFeatureFlags } from './useFeatureFlags'

const { user, login, logout } = useUserStore()
const { greeting } = useGreeting()
const username = ref('')

async function mockLoadFlags() {
  return { darkMode: true }
}

const { flags } = useFeatureFlags({
  loadMethod: mockLoadFlags,
  defaultFlags: { darkMode: false }
})

function loginUser() {
  if (username.value.trim()) {
    login(username.value)
  }
}
</script>

<style>
.dark {
  background: #333;
  color: #eee;
}
</style>
```


这种方法使得可组合和可扩展。无论你添加多少选项，调用者只需传递一个对象即可。


现在我们已经解决了状态共享、逻辑隔离和可组合配置问题。


# **4. 使用灵活参数处理不同输入类型**


有时，一个可组合函数接受一个参数，该参数可以是引用、计算值或普通字符串。


通常，你可能需要编写代码来处理每种情况。但是 `toValue()` 可以帮助自动解包这些。


我们也可以使用 `ref` 来执行相反的操作，但现在我们将专注于 `toValue()` 。


这种模式使可组合项更加灵活。你可以传递一个引用或一个字面量——这无关紧要。可组合项对它们一视同仁：


```javascript
import { computed, toValue } from 'vue'

export function useUserLabel(name) {
  return computed(() => {
    const actualName = toValue(name)
    return actualName ? `User: ${actualName}` : 'No user selected'
  })
}
```


我们依赖于 `toValue()` 来处理不同的输入形式。以下是我们可以如何使用它的方法：


```javascript
<script setup>
import { ref } from 'vue'
import { useUserStore } from './useUserStore'
import { useGreeting } from './useGreeting'
import { useFeatureFlags } from './useFeatureFlags'
import { useUserLabel } from './useUserLabel'

const { user, login, logout } = useUserStore()
const { greeting } = useGreeting()
const username = ref('')

async function mockLoadFlags() {
  return { darkMode: true }
}

const { flags } = useFeatureFlags({
  loadMethod: mockLoadFlags,
  defaultFlags: { darkMode: false }
})

// Here we pass a function returning user’s name
const userLabel = useUserLabel(() => user.value.name)

function loginUser() {
  if (username.value.trim()) {
    login(username.value)
  }
}
</script>

<template>
  <div :class="{ dark: flags.darkMode }">
    <div v-if="user.isLoggedIn">
      <p>{{ userLabel }}</p>
      <p>{{ greeting }}</p>
      <button @click="logout">Logout</button>
    </div>
    <div v-else>
      <input v-model="username" placeholder="Enter name" />
      <button @click="loginUser">Login</button>
    </div>
  </div>
</template>
```
