# 1.什么是API


API 代表**应用程序编程接口**。


从本质上讲，API 是一堆接受**输入**并提供可预测输出**的代码。**


![%E6%88%AA%E5%B1%8F2025-01-21_22.04.58.png](images/blog/2025-11-22-api/img-3a2e68de69.png)


可以将 API 视为一个**中间人**，它使应用程序能够进行交互，**而无需直接访问彼此的代码或数据库**。


您今天使用的几乎每一项数字服务（社交媒体、电子商务、网上银行、叫车应用程序）都是一堆协同工作的 API。


**例子：**

- **天气 API** – 如果您提供城市名称作为输入（ `"New York"` ），API 将返回**当前的温度、湿度和天气状况**。
- **Uber 乘车 API——**如果您提供**上车地址和目的地地址**，API 会找到**最近的可用司机**并计算预计车费。
- **Python 的**`sorted()` **API** – 如果您提供一个数字列表 ( `[5, 3, 8, 1]` )，API 将返回**排序后的列表**( `[1, 3, 5, 8]` )。

当工程师构建 API 时，他们会明确定义**API 接受哪些输入**以及**产生哪些输出**，从而确保不同应用程序之间的行为一致。


API 遵循简单的**请求-响应**模型：


![%E6%88%AA%E5%B1%8F2025-01-21_22.05.33.png](images/blog/2025-11-22-api/img-3f1bdcd417.png)

- 客户端（例如 Web 应用程序或移动应用程序）向 API 发出请求。
- API（托管在 API 服务器上）处理请求、与必要的数据库或服务交互并准备响应。
- API 以结构化格式（通常是 JSON 或 XML）将响应发送回客户端。

### **输入**


每个 API 都需要**特定类型的输入**，传递不正确的数据可能会导致错误。


例如：如果您尝试将您的姓名作为输入放入 Google Maps API 中，效果就不会很好。


一些 API 还**要求特定格式的输入**。


示例：**货币兑换 API**可能需要输入`"USD_TO_EUR"`而不是`"usd to euro"` 。


API 通常会在处理输入之前**验证输入**以确保其正确无误，这有助于保持**准确性和安全性**。


### **输出**


正如 API 需要**特定的输入**一样，它们也会返回**结构良好的输出**。


例如， **Google Maps API**始终**以相同的格式返回坐标**。


```plain text
{ "latitude": 40.6892, "longitude": -74.0445 }
```


如果 API 找不到该位置，它会提供错误响应并解释原因。


```plain text
{ "error": "Invalid address format", "code": 400 }
```

# **2. API 如何为现代应用程序提供支持**


您每天使用的应用程序（无论是**Gmail、Instagram、Uber 还是 Spotify** ）本质上**都是 API 的集合，顶部带有精致的用户界面（UI）** 。


大多数应用程序遵循**前端/后端架构**，其中：

- 后端由处理数据处理、业务逻辑和与数据库通信的**API组成**。
- 前端是一个与这些 API 交互的**图形用户界面 (GUI)** ，使应用程序变得用户友好且可访问，**而无需用户编写代码**。

![%E6%88%AA%E5%B1%8F2025-01-21_22.06.26.png](images/blog/2025-11-22-api/img-577ce75d8d.png)


让我们用一个现实世界的例子来解释这一点： **Uber** 。


### **后端**


在 Uber 应用程序以流畅、用户友好的体验出现之前，该公司首先构建了**支持叫车服务的核心 API：**

- 寻找附近的司机
- 计算票价和路线
- 处理付款
- 实时追踪
- 匹配乘客和司机

这些 API 在 Uber 的服务器上运行，构成了**后端基础设施**。每次您请求乘车、追踪司机或付款时，这些后端 API 都会处理请求。


**后端工程师**负责优化这些 API，改进乘车匹配算法，确保交易安全，并确保数百万用户的流畅体验。


### **前端**


后端 API 处理**所有复杂逻辑**，但它们**只能通过代码工作**— 这对于日常用户来说并不实用。这就是为什么公司在这些 API 之上构建**前端（用户界面）** ，让用户能够**直观地**与系统交互。


**示例：**当您输入上车和目的地地址时，前端会发送 API 请求以**查找附近的司机**并显示可用的车辆。


一旦行程完成，前端可能会调用处理支付API来显示收据。


# **3. API 的类型**


API 有不同的形式，取决于**谁可以访问它们**、**如何使用它们**以及**它们的用途**。


### **1.开放API（公共API）**


开放 API（也称为**公共 API** ）可供外部开发人员以极少的限制访问。


公司提供这些 API 是为了鼓励**第三方开发人员**集成他们的服务并在其基础上构建新的应用程序。


### **示例：YouTube 数据 API**


通常，当您使用**YouTube 应用**时，它会进行**内部 API 调用**来获取您的视频源、搜索内容或发表评论。但是，YouTube 还提供了一个**公共 API** ，允许开发人员**在应用之外**访问其中一些功能。


例如， **YouTube 搜索 API**允许开发者根据关键字获取视频结果。如果您向该 API 发送以`"machine learning tutorial"`作为搜索词的请求，它将返回一个结构化响应（JSON 格式），其中包含相关视频的列表，包括**标题、描述、缩略图和视频链接**。


这非常有用，因为它使开发人员能够在 YouTube 上构建自定义应用程序。


### **2. 内部 API（私有 API）**


**内部 API**又称为**私有 API** ，专为组织内部使用**而**设计。与开放 API 不同，外部开发人员无法访问这些 API，它们用于促进公司内部不同系统之间的无缝通信。


以**亚马逊**为例。当您下订单时，您可能会认为只有一个系统处理您的请求。实际上，**多个内部 API** （订单处理、库存、付款、物流等）在后台协同工作，以高效地完成您的订单。


每个 API 都**独立运行**，但它们通过明确定义的协议进行通信，以确保流程顺畅、高效。


内部 API 允许公司将其应用程序分解为**更小、更易于管理的服务**，从而更易于扩展。开发人员可以在不同的项目中**重复使用内部 API** ，从而减少**重复**并加快开发速度。


### **3. 代码接口**


我们讨论的前两种类型的 API（**开放 API 和内部 API** ）具有实用性，可用于**实际用例，**例如获取天气数据或预订行程。


但是还有另一类 API 是开发人员日常使用的：**代码接口**（也称为**库 API**或**编程 API** ）。


这些 API 并不连接不同的应用程序；而是在编程语言或框架内提供预定义的函数，以简化开发。


**示例：** Python 的内置列表 API


处理列表时，Python 提供了一组**内置函数（方法）来操作数据**。


```plain text
numbers = [5, 3, 8, 1, 4] numbers.sort() # API call to sort the list fruits = ["apple", "banana"] fruits.append("orange") # API call to add an element fruits.pop() # API call to remove the last element
```


开发人员不必从头编写排序算法，而是可以使用Python 中的`sort()`或`sorted()` 。


代码 API 不仅限于内置的编程语言函数。以AI/ML 库**TensorFlow**为例。它提供了一个**高级 API，**用于训练机器学习模型，而无需从头开始实现复杂的数学运算。


例如，使用 TensorFlow 的 API 创建**神经网络**非常简单：


```plain text
import tensorflow as tf model = tf.keras.Sequential([tf.keras.layers.Dense(64, activation="relu")])
```


编程 API 抽象了复杂性，以便开发人员可以专注于构建解决方案，而不是重新发明轮子。

# **4. API 通信方式**


API 使用不同的**协议和架构**进行通信，这些协议和架构定义了如何发送请求、如何格式化响应以及如何在系统之间交换数据。


### **1. REST（表述性状态转移）**


REST 是目前使用最广泛的 API 通信方法。它**轻量、无状态、可扩展**，非常适合 Web 服务和移动应用程序。


REST API 遵循一组设计原则并使用**HTTP 方法**（GET、POST、PUT、DELETE）执行操作。


REST API 基于**资源**，每个资源都通过**URL（端点）**访问。API 遵循**客户端-服务器模型**，这意味着客户端发送请求，服务器处理该请求并发送响应。


### **示例：书店的 REST API**


**检索书籍列表（GET 请求）：**


```plain text
GET https://api.bookstore.com/books
```


**响应（JSON）：**


```plain text
[ { "id": 1, "title": "Clean Code", "author": "Robert C. Martin" }, { "id": 2, "title": "The Pragmatic Programmer", "author": "Andrew Hunt" } ]
```


### **2.SOAP（简单对象访问协议）**


SOAP 是一种较旧的 API 通信方法，**依赖于基于 XML 的消息传递**。


与轻量级的 REST 不同，SOAP 更加结构化且更安全，非常适合银行、医疗保健和企业应用程序。


SOAP 消息使用**XML 格式**发送，需要**WSDL（Web 服务描述语言）文件**，该文件定义了 API 的可用功能和请求结构。


### **示例：银行服务的 SOAP API**


**请求：**获取账户余额


```plain text
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/ " xmlns:bank="http://bank.example.com/ "> <soapenv:Header/> <soapenv:Body> <bank:GetAccountBalance> <bank:accountNumber>123456</bank:accountNumber> </bank:GetAccountBalance> </soapenv:Body> </soapenv:Envelope>
```


**回复：**


```plain text
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/ "> <soapenv:Body> <bank:GetAccountBalanceResponse> <bank:balance>5000.00</bank:balance> </bank:GetAccountBalanceResponse> </soapenv:Body> </soapenv:Envelope>
```


### **3. GraphQL**


GraphQL 是 REST 的替代方案，它**允许客户端精确请求所需的数据**，从而使其对现代应用程序来说更加高效。与需要多次 API 调用才能获取相关数据的 REST 不同，GraphQL 可以**在单个请求中获取所有必要的数据**。


GraphQL 没有预定义端点，而是公开单个**API 端点**，客户端发送查询来请求特定字段。


### **示例：在单个请求中获取用户的个人资料及其最近的帖子。**


```plain text
{ user(id: 123) { name email posts { title likes } } }
```


回复：


```plain text
{ "data": { "user": { "name": "Alice", "email": "alice@example.com ", "posts": [ { "title": "Hello World", "likes": 100 }, { "title": "GraphQL is Amazing!", "likes": 200 } ] } } }
```


### **4. gRPC**


gRPC（Google Remote Procedure Call）是一种**高性能 API 通信方式**，它使用**协议缓冲区（Protobuf）**代替 JSON 或 XML，从而更快、更高效。


gRPC 使用**二进制数据格式**而不是基于文本的格式，从而减少了有效负载大小，并且支持**双向流**，这意味着客户端和服务器可以同时发送数据。

# **5. 如何使用 API（分步指南）**


使用 API 乍一看可能很复杂，但它遵循简单的**请求-响应**模式。


以下是有关如何逐步**查找、访问和与 API 交互**的指南：


### **步骤 1：查找要使用的 API**


在使用 API 之前，您需要**确定适合**您需求的 API。API 可用于不同的服务，例如天气数据、金融、社交媒体等。

### **第 2 步：阅读 API 文档**


API 文档解释了**如何使用 API、可用端点、身份验证和响应格式**。


**示例：** [**OpenWeatherMap**](https://substack.com/redirect/3993c587-0088-487b-9afb-8c955cacafe5?j=eyJ1IjoiNG56aGx0In0.ubDd-vE5RRY545Zu5Ix1zUh-lZfjyjPKWGBGKFEw4vU) API


OpenWeatherMap API 允许用户获取实时天气数据。以下是其主要组件的细分：


**API 网址**


```plain text
https://api.openweathermap.org/data/3.0/weather?q=city_name&appid=YOUR_API_KEY
```


**必需参数：**

- `q` ：城市名称（例如， `London` ）
- `appid` ：API 密钥（访问所需）

### **步骤 3：获取 API 访问权限（API 密钥/身份验证）**


大多数 API 都**需要身份验证**，以防止未经授权的访问并管理使用限制。


### **常见的身份验证方法：**

- **API 密钥 -** API 服务提供的唯一密钥
- **OAuth 2.0-**通过 Google、Github 等安全登录。
- **JWT（JSON Web Token）：**基于 Token 的身份验证
- **基本身份验证：**用户名+密码（Base64编码）

**示例：获取 API 密钥 (OpenWeather API)**

- 在[https://home.openweathermap.org/users/sign_up](https://home.openweathermap.org/users/sign_up)注册。
- 转到**API 密钥**部分并生成密钥。
- 在请求中使用 API 密钥：

```plain text
GEThttps://api.openweathermap.org/data/2.5/weather?q=London&appid=YOUR_API_KEY
```


### **步骤 4：使用 Postman 或 cURL 测试 API**


在编写代码之前，**先测试 API**以查看其响应情况。


### **选项 1：使用 Postman（推荐给初学者）**

- 下载并安装[**Postman**](https://substack.com/redirect/a04ab894-10c2-4f75-b6d0-33a61d27f1d6?j=eyJ1IjoiNG56aGx0In0.ubDd-vE5RRY545Zu5Ix1zUh-lZfjyjPKWGBGKFEw4vU) 。
- 点击**“新请求”** ，输入 API 端点 URL（ [`https://api.openweathermap.org/data/3.0/weather?q=London&appid=YOUR_API_KEY`](https://api.openweathermap.org/data/3.0/weather?q=London&appid=YOUR_API_KEY) ）。
- 选择**GET**作为 HTTP 方法。
- 单击**“发送”**并查看**JSON 格式**的响应。

### **选项 2：使用 cURL（适用于命令行用户）**


您还可以使用 cURL 直接**从命令行**测试[**API**](https://substack.com/redirect/76337fad-8814-4a38-8e2f-e346027bce06?j=eyJ1IjoiNG56aGx0In0.ubDd-vE5RRY545Zu5Ix1zUh-lZfjyjPKWGBGKFEw4vU) 。


```plain text
curl -X GET "https://api.openweathermap.org/data/3.0/weather?q=New+York&appid=YOUR_API_KEY "
```


### **步骤 5：编写代码来调用 API**


现在您已经测试了 API，是时候将**其集成到您的应用程序中了**。


### **示例：在 Python 中调用 API**


```plain text
import requests url = "https://api.openweathermap.org/data/3.0/weather?q=New York&appid=YOUR_API_KEY" response = requests.get(url) if response.status_code == 200: data = response.json() print(f"Temperature: {data['main']['temp']}°C") else: print("Error:", response.status_code)
```

- `requests.get(url)` – 发送 API 请求。
- `response.json()` – 将响应转换为 JSON。
- `if response.status_code == 200` – 检查请求是否成功。

### **步骤 6：处理错误和速率限制**


API**并不总是能返回完美的响应**。您应该处理：

- **输入无效**（例如，错误的城市名称）。
- **身份验证错误**（例如，API 密钥过期）。
- **速率限制**（例如超出请求限制）。

### **示例：处理 Python 中的 API 错误**


```plain text
if response.status_code == 200: data = response.json() print(f"Weather: {data['weather'][0]['description']}") elif response.status_code == 401: print("Error: Invalid API key") elif response.status_code == 404: print("Error: City not found") else: print(f"Unexpected error: {response.status_code}")
```


### **步骤 7：在应用程序中使用 API 响应**


一旦从 API 获取数据，就可以**在网络或移动应用程序中动态显示它**。


**示例：**您可以使用 OpenWeatherMap API 构建天气仪表板。

- 从 API 获取实时天气数据。
- 解析并提取相关细节（温度、湿度、条件）。
- 以用户友好的格式显示天气报告。
