# **1.** [**Webcrumbs**](https://tools.webcrumbs.org/frontend-ai) **- Instant Component Generation with TailwindCSS**


Webcrumbs 的 Frontend AI 是一款 AI 助手，旨在通过根据用户请求、图像或屏幕截图生成 UI 组件代码来帮助前端开发人员加快开发过程。


这个AI工具允许开发者编写特定的提示，生成UI，并生成相应的JSX代码。令人兴奋的是，代码是使用 Tailwind CSS 或常规 CSS 生成的，可以轻松复制并粘贴到您的项目中。


观看[演示视频](https://www.youtube.com/watch?v=HO3BC3pP1KE&t=93s)或阅读文档以更好地理解！


**其主要特点是：**


向人工智能询问任何组件、元素或功能。它将生成组件的 UI 以及代码。还值得一提的是，它允许您与其他人共享这些组件。您可以[在此处](https://tools.webcrumbs.org/frontend-ai?hash=7d50554e286b1162b7b69f0a97cb4088465e2638a11bcc744fa5fe92b31a3050&theme=2997f38b0e8df0df054683159c8f446b98f2003d443347da8622f598be14d9f9)与我们使用 Webcrumbs 前端 AI 生成的组件进行交互，甚至可以使用它。


需要注意的一件事是 Webcrumbs 与框架无关。它允许您在构建应用程序时选择最适合使用的框架。


通过调整字体、颜色、间距和其他元素来定制您的设计，以满足您的项目的要求。


另一个改变游戏规则的功能是，这个AI助手可以让你提供Figma组件、图片和截图，然后它会为你生成JSX代码。你再也不需要害怕与疯狂的设计师合作了。


它还提供实时协作、语法突出显示、版本控制支持、集成调试工具等等。


# **2.** [**Watsonx Code Assistant**](https://www.ibm.com/products/watsonx-code-assistant) **- 企业级代码生成和自动化**


Watsonx 由 IBM 团队开发，是一款由 IBM Granite[基础模型](https://www.ibm.com/products/watsonx-ai/foundation-models)提供支持的编码助手，该模型使用生成式 AI 来加速开发，同时保持安全性。它利用人工智能根据自然语言请求或现有源代码生成的建议来生成高质量代码。


**它的一些独特功能是：**

- 根据自然语言请求生成具有适用语法的新代码。
- 通过向您显示生成的代码建议的来源来提高透明度。
- 重构遗留代码或将其翻译成另一种编程语言。

Watsonx Code Assistant for Red Hat Ansible Lightspeed 使用生成式人工智能来加速 Ansible Playbook 的生成，并帮助实现 IT 自动化。它允许您以自然语言传达指令，并以人工智能生成的内容推荐进行响应。


该工具将 AI 内容推荐直接集成到您的 IDE 中，并根据现有 Playbook 内容提供个性化代码建议，从而提高工作效率。它还提供了训练数据源的透明度，并为每个生成的任务提供了详细的解释。


IBM Watsonx Code Assistant for Z 风险较低且成本较低，是一种生成式 AI 辅助解决方案，旨在减少现代化过程并延长大型机应用程序的生命周期。如果您想改进现有的应用程序，那么这是适合您的工具。它通过分析功能和应用程序发现、代码解释、自动代码重构、代码优化建议、COBOL 到 Java 转换和验证来支持端到端应用程序开发人员生命周期。


# **3.** [**Coderabbit**](https://coderabbit.ai/) **- 团队代码生成和错误检测**


这是一个非常独特的工具。事实上，它是 GitHub 和 GitLab 上安装最多的 AI 应用程序，审查了超过 300 万个拉取请求，审查了 50 万多个存储库。


Coderabbit 是为您和您的团队提供的人工智能代码审查器。它可以生成人工智能驱动的上下文反馈并更快地捕获可能已进入生产环境的错误，从而帮助减少代码审查时间。


这是它的工作原理！它以所做更改的摘要开始审核，并将其分解为易于理解的要点。然后，它会生成拉取请求的技术演练，其中包含更改内容的完整详细信息。它能够建议可能相关的问题，并提供更改的流程图。


Coderabbit 会审核您的代码更改，并在 GitHub 或 GitLab 上的代码审核中为您提供“一键式”解决方案。


**它的一些主要特点：**

- 审稿意见实时聊天，让您可以与AI助手聊审稿意见甚至可能的解决方案。
- 带有序列图的拉取请求摘要。

# **4.** [**v0**](https://v0.dev/faq) **- 使用 React 和 TailwindCSS 进行智能 UI 合成**


[Vercels](https://vercel.com/)的 AI 团队创建了[v0](https://v0.dev/chat)作为前卫的 AI 助手，特别是针对前端开发人员，主要关注 React、Next.js App Router 和现代 Web 开发实践。尽管它目前处于公开测试阶段，但它旨在通过提供清晰的编码解决方案和解释来模仿高级开发人员。


**其一些主要功能：**

- 多种编程语言：虽然v0的主要焦点是JavaScript/TypeScript和React生态系统，但它也提供了对多种语言和框架的支持
- UI/UX设计实现：v0可以将上传的图像翻译成React（TypeScript）代码。它还可以根据 UI 的文本描述生成代码，并根据最佳实践提出用户体验的改进建议。
- UI 和代码生成：v0 可以使用 TailwindCSS 类生成响应式 UI 组件并生成 UI 代码。它还可以生成系统架构等图表、可视化数据流，并利用 Mermaid 图表语言为算法创建流程图。

v0 不是开源的，但提供了公平的免费套餐。


# **5.** [**Code Llama**](https://about.fb.com/news/2023/08/code-llama-ai-for-coding/) **- 用于代码生成和调试的人工智能**


Code Llama 是一款基于[Llama 2](https://about.fb.com/news/2023/07/llama-2/)构建的 AI 助手，它使用文本提示来生成和讨论代码。


该工具由 Meta AI 团队构建，支持 Java、Python、C++、TypeScript 等流行语言。


Code Llama 发布时具有三个参数，经过[中间填充](https://arxiv.org/abs/2310.09930)(FIM) 功能的训练，允许它们将代码插入到现有代码中，这意味着它们可以帮助完成代码和调试等任务。


# **6.** [**CodeParrot**](https://www.codeparrot.ai/) **- UI 组件的设计到代码人工智能**


如果您想快速构建令人惊叹的应用程序 UI，那么 CodeParrot 就是您的完美工具。永远不要再吝惜良好的用户界面！


CodeParrot 是一个 vscode 插件，它使用 AI 将 Figma 组件或屏幕截图转换为代码。它为您提供了使用_GitHub 或 Figma_登录的选项。 Figma 登录选项可让您直接浏览 Vscode 中 Figma 文件中的组件。


它支持许多编程语言和框架，包括开箱即用的支持、React、Tailwind 和 TypeScript。您可以从[VS Code 市场](https://marketplace.visualstudio.com/items?itemName=CodeParrot-ai.codeParrot)安装 CodeParrot 插件。


**其一些主要功能：**

- 它根据提示生成代码，并允许您自定义设置以选择生成代码所用的语言。
- 它增加了您现有的工作流程。
- 它按照您的编码标准生成代码

可以阅读[文档](https://codeparrot.ai/docs/quick-start)。您将找到有关使用 CodeParrort 构建简单组件和完整屏幕的教程。您还可以观看[简短的演示](https://www.youtube.com/watch?v=3f44DY-xq4Q&t=1s)以更好地了解该工具。


不是开源的，但它提供每月大约 10 个请求的免费版本。


# **7.** [**MutableAI**](https://mutable.ai/) **- AI重构和代码优化**


人工智能编码助手和LLM都是很棒的工具，但他们的回应和建议很笼统。可变人工智能的构建方式有所不同。其目标是将开发人员的生产力和满意度提高 10 倍。


它可以帮助您为代码编写文档。当您将代码推送到存储库时，Mutable AI 将为您的存储库编写一篇维基百科文章。本文使用直接指向您的代码的图表和引用来记录您的存储库。这样，您就不必为编写代码文档而苦苦挣扎；专注于真正重要的事情：编写代码。


**它的一些独特功能：**

- 当默认分支有合并拉取请求 (PR) 时，Mutable.ai 会通过准备新版本的 wiki 文章来自动更新 wiki 文章。
- MutableAI 不会让您被排除在外。它会创建这些更改的简短摘要，并每周通过电子邮件将其发送给订阅用户。
- MutableAI 提供了一项功能，使您能够为其 AI 提供额外的上下文，以修改生成的 wiki 文章中可能低估了关键行为的部分文本。
- 通过人工智能聊天和整个团队的非技术模式快速提取答案。

它不提供免费版本，但您可以尝试一下。


# **8.** [**ellipsis.dev**](https://www.ellipsis.dev/) **- 拉取请求审查和代码生成**


Ellipsis 是一个 AI 助手，可以审查拉取请求（特别是基于逻辑正确性）、创建发行说明并直接在 GitHub 上修复错误。


如果您经常忘记为拉取请求添加描述，或者不知道为每次提交添加什么内容，请不要担心；省略号会为你做到这一点。这就像有一双额外的眼睛来发现问题，并有额外的双手来解决问题。


它还允许您添加规则（以自然语言），以说明 Ellipsis 在审查您的代码时应检查的内容。这有助于定制评论。


Ellipsis 提供对 20 多种语言的支持，每天在超过 29K 的在线代码库中审查超过 2.1K 条评论。


# **9.** [**CodeT5+**](https://blog.salesforceairesearch.com/codet5-open-code-large-language-models/#:~:text=As%20an%20encoder%2Ddecoder%20model,the%20quality%20of%20model%20outputs.) **- 开源代码生成和理解**


CodeT5+是基于[T5架构的代码理解和生成的AI助手。](https://huggingface.co/docs/transformers/en/model_doc/t5)它经过跨度文本代码匹配、去噪、因果语言建模和对比学习的训练，是原始[CodeT5 系列](https://arxiv.org/abs/2109.00859?ref=blog.salesforceairesearch.com)的高级版本。


它可以充当仅编码器、仅解码器或编码器-解码器模型，并且可以轻松适应许多编码任务。


另一个令人着迷的功能是 CodeT5+ 可以有机地用作通过检索增强的端到端代码生成系统。


**以下是其三大特点：**

- 根据自然语言描述生成代码。
- 如果给定一个函数名，它可以完成一个功能块的整个代码。
- 使用自然语言描述生成函数的摘要。

# **10.** [**Jam.dev**](https://jam.dev/) **- 一键错误报告**


该工具可将错误报告时间缩短 20 倍。那真是难以置信。不再浪费时间报告错误。


Jam 是一个浏览器扩展，只需单击两次，您就可以以最适合工程师的方式报告错误。报告错误时，它会自动包含导致错误发生的最后 30 秒、网络请求以及有关会话的所有元数据，包括网络速度。然后，当修复错误的工程师打开链接时，他们就拥有有关该错误的完整信息，可以快速找出问题所在并轻松修复。


您所需要做的就是录制视频、截图或捕捉即时重播，剩下的工作就交给 Jam 吧。


Jam 是第一个专门为报告错误而设计的屏幕录像工具，它可与所有问题跟踪软件配合使用，包括 Jira、Linear、Asana 等。


专门为帮助开发人员针对收到的 Jam bug 报告进行更快的调试和代码修复而构建的 AI 调试助手。


JamGPT 接收来自 Jam 的数据，使其能够在错误报告期间访问所有浏览器上下文。


JamGPT 可以：

- 确定所报告错误的根本原因并提出可能的原因。
- 与开发人员互动并根据对话完善其建议。
- 修复导致错误的代码
