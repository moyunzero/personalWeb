### 1. JSM Skills (JavaScript-Mastery-Pro/skills)


**描述**：JSM Skills 是 JS Mastery 工作流背后的 Agentic Development 技能集。AI Agent 强大但无状态，容易自信地构建错误内容；这些技能通过五条 Slash 命令赋予 Agent 工程纪律，包括会话间记忆、构建前的架构思考和构建后的结构化审查。


**如何使用**：通过 `npx skills@latest add JavaScript-Mastery-Pro/skills` 安装，支持任何兼容 [SKILL.md](http://skill.md/) 格式的 Agent（如 Claude Code、Cursor 等）。


**使用场景**：大型功能开发、UI 组件库构建、跨多会话的长期项目。


**优势**：通过 `/architect`（规划）、`/remember`（记忆）、`/review`（审查）、`/recover`（问题恢复）、`/imprint`（UI 一致性）等命令，形成清晰的工程循环，让开发者始终掌握方向，避免 Agent 失控。


**扩展性**：全部基于纯 Markdown 实现，易于 Fork、定制命令或贡献新技能。


### 2. Superpowers (obra/superpowers)


**描述**：一个可组合的 Agentic 技能框架与实用的软件开发方法论。从会话开始，Agent 不会直接写代码，而是先澄清目标、制定规范、生成实现计划，并通过子 Agent 驱动开发流程。


**如何使用**：根据不同 Agent（如 Claude Code、Cursor 等）通过插件市场或命令安装，会话启动时自动激活。


**使用场景**：从规范到交付（Spec-to-Ship）的复杂功能实现、需要高质量保障的长期项目。


**优势**：强调真实红绿 TDD、YAGNI 和 DRY，支持子 Agent 自主工作数小时仍不偏离计划，显著提升输出可靠性和工程质量。


**扩展性**：技能高度可组合，文档详尽，支持多个平台，并有商业支持选项。


### 3. GSD-Core (open-gsd/gsd-core) — Git. Ship. Done.


**描述**：Git. Ship. Done. 的核心框架，提供全面的 Agentic 开发工具、技能、Hook 和标准化流程。


**如何使用**：在支持的 Agent 中安装激活，结合内置技能和上下文管理工具使用。


**使用场景**：端到端项目生命周期管理、团队协作与高速迭代环境。


**优势**：内置质量门控、动态上下文管理和实用交付流程，帮助团队高效“Get Shit Done”。


**扩展性**：模块化架构，支持插件、自定义技能，社区活跃且持续演进。


### 4. CodeGraph (colbymchenry/codegraph)


**描述**：预索引、自动同步的代码知识图谱，为 Claude Code、Cursor、Codex 等 Agent 提供语义智能。100% 本地运行。


**如何使用**：CLI 安装（`curl` 或 `npm`），运行 `codegraph install` 连接 Agent，再 `codegraph init` 初始化项目（自动同步文件变更）。


**使用场景**：大型或多语言代码库、架构问题、变更影响分析、重构任务。


**优势**：提供外科手术式的精准上下文，大幅减少工具调用次数（基准测试约 58% 更少）和 Token 消耗，优于传统 grep/逐文件读取。


**扩展性**：支持多种主流 Agent，未来平台功能可扩展至 PR 级别洞察，适合个人与团队使用。


### 5. Understand Anything (Egonex-AI/Understand-Anything)


**描述**：将任意代码库、知识库或文档转化为交互式知识图谱，支持探索、搜索和提问。目标是“会教人的图谱，而非炫耀复杂的图谱”。


**如何使用**：安装插件后，执行 `/understand` 命令生成图谱与仪表盘。


**使用场景**：大型项目快速上手（onboarding）、业务逻辑映射、影响分析、知识库导航。


**优势**：支持结构图、领域视图、引导式游览、语义搜索和角色自适应界面，帮助开发者真正理解系统而非仅浏览代码。


**扩展性**：支持多 Agent 平台、多语言与领域定制，插件架构灵活。


### 6. Awesome Design MD (VoltAgent/awesome-design-md)


**描述**：从流行品牌和设计系统中提取的 [DESIGN.md](http://design.md/) 文件合集。[DESIGN.md](http://design.md/) 是 Google Stitch 提出的纯文本设计规范格式。


**如何使用**：将合适的 [DESIGN.md](http://design.md/) 放入项目根目录，指示 Agent “按照此设计语言构建 UI”。


**使用场景**：跨项目 UI 一致性维护、快速品牌化原型开发、设计系统落地。


**优势**：Markdown 格式对 LLM 极其友好，无需 Figma/JSON 转换，Agent 可直接生成高一致性的视觉输出。


**扩展性**：社区驱动的集合，支持请求自定义 [DESIGN.md](http://design.md/)，与其他 UI 技能完美结合。


### 7. ECC (affaan-m/ECC)


**描述**：Agent 工作台性能优化系统，聚焦技能、本能、记忆、安全和研究优先开发。


**如何使用**：安装对应 Agent 的插件/配置，结合规则、技能和内存管理功能。


**使用场景**：企业级项目、安全敏感工作流、多 Agent 编排、性能优化需求。


**优势**：强化安全机制（如 GateGuard）、持久化记忆，并针对不同 LLM/平台进行优化。


**扩展性**：丰富的插件、Manifest 和研究组件，支持广泛平台与深度自定义。


### 8. GStack (garrytan/gstack)


**描述**：Garry Tan 的意见化 Claude Code 设置，包含 23+ 专业角色工具（CEO、设计师、工程经理、QA 等）。


**如何使用**：安装激活后，按需调用不同角色工具。


**使用场景**：全栈产品开发、复杂项目管理、自主 Agent 协同。


**优势**：提供从战略规划到执行、测试、发布的完整生命周期覆盖，像拥有一整个专业团队。


**扩展性**：模块化设计，持续更新，欢迎社区贡献新角色或功能。
