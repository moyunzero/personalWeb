浏览器的渲染过程是一个复杂的多个线程、进程和阶段的协调，将原始的 HTML、CSS 和 JavaScript 转换为屏幕上的交互式像素。虽然浏览器的渲染过程并不新鲜，并且关于这个主题的信息非常丰富，但我认为这足以写一篇互动文章。所以，本质上，我们的目标是理解在这两个点之间发生了什么：

1. 在浏览器中输入一个 URL 并按回车键。
2. 网站已显示在屏幕上。

# **导航和网络**


该过程从你在浏览器地址栏中输入一个 URL 并按回车键开始。让我们假设我们正在尝试加载 _**`https://example.com`**_ 。浏览器不知道从哪里获取这个网站的内容。这只是我们人类能理解的名字。浏览器需要将这个名称转换为 IP 地址。这就是 DNS 查找发挥作用的地方。

> DNS 代表域名系统。该系统使用指定的授权域名服务器将域名映射到数字 IP 地址。

![%E6%88%AA%E5%B1%8F2024-11-14_22.40.28.png](images/blog/2024-12-14-36ddf5c0/img-38bc452a89.png)


查询之旅包括将您输入的域名转换为 IP 地址所需的每个步骤。查询的第一站是递归服务器，然后它联系一系列权威域名服务器将域名转换为 IP 地址。最终，返回请求域名的 IP 地址。

<details>
<summary>什么是域名服务器？</summary>

域名服务器是用于查找与域名关联的 IP 地址的专用服务器。您可以在域名的 DNS 记录中找到这些域名服务器。


</details>


因此，现在我们有了 IP 地址，浏览器就可以与托管网站的服务器建立连接。这是通过 TCP 握手过程完成的。


# **TCP/TLS 握手**


TCP（传输控制协议）握手是一个三步过程，用于在客户端和服务器之间建立连接。它包括客户端和服务器之间交换的一系列数据包，以确保可靠的连接。这三个步骤是：

- SYN：客户端向服务器发送一个同步（SYN）数据包以建立连接。
- SYN-ACK：服务器向客户端发送确认（SYN-ACK）数据包。
- ACK：客户端向服务器发送确认（ACK）以完成握手。

此时，我们只与服务器建立了连接。下一步是确保这个连接的安全。记得 URL 旁边的 _**`https`**_ 锁形图标吗？它定义了一个安全连接，并使用另一种握手方式完成，称为 TLS 握手，涉及几个步骤。但在我们深入之前，如果你能理解一些与 TLS 握手相关的术语，那就更容易了。

> **Cipher**  - 密码是一种用于加密和解密数据的算法。它用于确保互联网上数据传输的安全性。  
> **Certificate**- 证书是一个包含网站身份和公钥信息的文件。它用作数字签名，以验证网站的合法性。  
> **Certificate Authority** (CA)-证书颁发机构是一个受信任的实体，它向网站颁发数字证书。它验证网站的标识并确保证书的真实性。  
> 公钥 - 公钥用于在发送到互联网之前加密数据。它与客户端共享以建立安全连接。  
> 私钥 - 私钥用于解密从客户端接收到的数据。它被保密，不会与任何人共享。

TLS 握手过程有多个步骤。我已经将它们分为 4 个主要类别：

- **Hello Messages**：客户端发送一个 _**`ClientHello`**_ 消息以启动握手，服务器响应一个 _**`ServerHello`**_ 消息以及其数字证书。
- 密钥交换：服务器可能发送密钥交换信息，客户端则回复自己的密钥交换消息（以及如果需要则证书）。
- 密钥规范：客户端发送一个 _**`CipherSpec`**_ 消息以切换到协商的加密方式，随后发送一个 Finished 消息
- 服务器最终化：服务器回复自己的 _**`CipherSpec`**_ 和 Finished 消息，完成握手并建立安全连接。
> SSL，或称安全套接字层，是专为 HTTP 开发的原始安全协议。SSL 已被 TLS，或传输层安全性协议，所取代。SSL 握手现在被称为 TLS 握手，尽管“SSL”这个名字仍在广泛使用。

服务器仅确认了连接并建立了安全通道。浏览器（客户端）在幕后处理所有这些协商过程。只有在 TLS 握手完成且双方交换了其完成消息后，他们才能开始安全地发送和接收加密内容。


# **HTTP 请求/响应周期**


现在浏览器请求网站内容。服务器处理这个请求并发送回响应。这就是 HTTP 请求/响应周期发挥作用的地方。后端开发者负责处理请求，而前端开发者对这个周期的响应部分更感兴趣。


![%E6%88%AA%E5%B1%8F2024-11-14_22.48.21.png](images/blog/2024-12-14-36ddf5c0/img-93c060a2d5.png)


TTFB 代表首次字节时间。这是服务器发送响应客户端请求的第一个字节所需的时间。它是衡量服务器响应时间和网站性能的重要指标。较低的 TTFB 表示更快的服务器响应时间。s

> 好的 TTFB 值是 0.8 秒或更少，较差的值大于 1.8 秒。介于两者之间的值属于“需要改进”。

# **Tokenization** 


浏览器将开始解析这个 HTML 响应。它读取 HTML 的原始字节，并根据文件的指定编码将它们转换为单个字符，如 _**`UTF-8`**_ 。以下是一个字节可能看起来像的例子：


```javascript
01001000 01010100 01001101 01001100
```

> 每组 8 位代表一个字节，每个字节可以根据编码标准被转换为一个字符、符号或指令。  
> 例如，在 UTF-8 编码中，字母 A 的二进制序列是 01000001。字母 B 是 01000010

浏览器将字符序列转换为不同的标记并将它们分组，例如， _**`<html>`**_ ， _**`<body>`**_ — 以及其他位于尖括号内的字符串。每个标记都是一个有意义的单元，代表一个 HTML 组件。


![%E6%88%AA%E5%B1%8F2024-11-14_22.50.15.png](images/blog/2024-12-14-36ddf5c0/img-449c0337ea.png)


# **DOM 树创建**


DOM（文档对象模型）树是 HTML 文档结构的层次表示。它将元素表示为节点树。每个节点代表文档中的一个元素、属性或文本内容。类似于这样：


```javascript
<html lang="en-US">
  <head>
    <style>
        .heading {
            color:#0099ff; 
            font-size: 14px
        }
        p {
            margin: 0.5em 0;
        }
        a {
            color: #0099ff; 
            text-decoration: underline;
        }
    </style>
  </head>
  <body>
    <h1 class="heading">My Page</h1>
    <p>A paragraph with a <a href="https://example.com/about">link</a></p>
  </body>
</html>
```


DOM 是一个树状结构，因此可以轻松显示父子关系。


每次浏览器渲染网页时，它都会经历一个多步骤的过程：将 HTML 字节解析为字符，识别标记，将它们转换为节点，最后构建 DOM 树。虽然 DOM 树定义了 HTML 元素的结构和关系，但它并不指定它们的视觉外观。这正是 CSSOM 的作用。


#  **CSSOM 树创建**


CSS 对象模型（CSSOM）是应用于 HTML 文档的 CSS 样式的表示。它与 DOM 树类似，但表示的是 CSS 样式而不是 HTML 结构。CSSOM 用于计算文档中每个元素的最终样式。


![%E6%88%AA%E5%B1%8F2024-11-14_22.51.36.png](images/blog/2024-12-14-36ddf5c0/img-8c264d1d17.png)


浏览器将这些样式解析为一种流线型、内存高效且优化的数据结构。这种结构旨在高效组织样式规则，允许根据匹配选择器快速检索和应用样式。


一旦 DOM 和 CSSOM 构建完成，浏览器就可以开始创建渲染树。


# **渲染树创建**


渲染树是 DOM 树和 CSSOM 树的组合。它表示页面的视觉结构，包括布局和样式信息。渲染树用于计算布局并在屏幕上绘制元素。


它为每个元素提供了计算后的样式和布局信息，使浏览器能够准确渲染页面。这使我们来到了渲染过程的最后阶段 - 布局。


# **布局**


您可能已经听说过在 CSS 上下文中提到的术语 _**`layout`**_ 。它是计算页面中每个元素的确切位置和大小的过程。布局过程包括根据渲染树确定每个元素的尺寸、边距、填充、边框和定位。您可能已经看到过重叠的元素，例如模态窗口、下拉菜单等。这就是布局过程发挥作用的地方。


即使 HTML 标记顺序正确，由于 CSS 属性改变了布局，元素也不一定按照它们被书写的顺序出现。


布局步骤的结果是显示每个元素的精确计划，随后是绘制步骤，其中视觉表示将在屏幕上呈现。


## **绘画**


在绘画阶段，浏览器获取结构化布局信息，并将每个元素绘制到屏幕上。这个过程填充颜色并应用图像、边框、阴影和其他视觉样式。绘制顺序基于堆叠上下文，确保元素根据 z-index 和其他属性正确分层。


绘画过程已优化以最小化需要重绘的像素数量。浏览器使用分层、合成和缓存等技术来高效渲染页面。
