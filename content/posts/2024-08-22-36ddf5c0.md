## 防抖


一个众所周知的防抖使用案例是预测搜索。


想象一下，你正在为一个拥有数千种产品的电子商务网站构建搜索功能。当用户尝试搜索某些内容时，您的应用程序将调用API来获取与用户查询字符串匹配的所有产品。


```css
const handleKeyDown = async (e) => {
 const { value } = e.target;
 const result = await search(value);
 // set the result to a state and then render on UI
}

<Input onKeyDown={handleKeyDown} />
```


这种方法看起来不错，但是有一些问题：

1. 每次按键事件都会发起一个 API 调用。如果用户输入了 15 个字符，那对于一个用户来说，就会有 15 个 API 调用。这是无法扩展的。
2. 当这15个 API 调用的结果到达时，你只需要最后一个调用的结果。之前14个调用的结果将会被丢弃。这会消耗用户大量的带宽，而且在网络较慢的用户面前会出现明显的延迟。
3. 在用户界面上，这15个 API 调用会触发重新渲染。这会导致组件变得迟钝。

解决问题的方法是防抖处理。


基本思路是等待用户停止输入，再将延迟进行API调用。


```css
const debounce = (fn, delay) => {
 let timerId;
 return function(...args) {
  const context = this;

  if (timerId) {
    clearTimeout(timerId);
  };
  timerId = setTimeout(() => fn.call(context, ...args), delay);
 }
}

const handleKeyDown = async (e) => {
 const { value } = e.target;
 const result = await search(value);
 // set the result to a state and then render on UI
}

<Input onKeyDown={debounce(handleKeyDown, 500)} />
```


防抖函数是一个通用的实用函数，需要两个参数：

- fn：将延迟的函数调用。
- delay：延迟时间，以毫秒为单位。

在函数内部，使用setTimeout来延迟实际的函数调用（fn）。如果在定时器结束之前再次调用fn，则定时器会被重置。


通过更新实现，即使用户输入了15个字符，我们也只会进行1次API调用（假设每次按键的响应时间都小于500毫秒）。这解决了在构建该功能时遇到的所有问题。


在生产代码库中，不需要编写自己的防抖实用函数。可以使用了类似lodash这样的JS实用程序库，其中包含了这些方法。


## 节流


防抖动在性能方面非常好，但在某些情况下，我们不希望在 x 秒之后才被通知到更改。


比如说正在构建一个类似谷歌文档或 Figma 的协作工作空间。其中一个关键功能是用户应该能够实时了解其他用户所做的更改。


到目前为止，我们只知道两种方法：


1. 新手法：每当用户移动鼠标指针或输入内容时，都发起一次 API 调用。


2. 防抖法：它确实解决了性能问题，但从用户体验的角度来看，它很糟糕。你的同事可能会写一个有300个字的段落，而你只在最后一次才得到通知。这种情况还算是实时吗？


这就是节流法的作用。它位于上述两种方法的中间。基本思路是按照一定的时间间隔通知 - 不是在结束时，也不是在每次按键时，而是定期通知。


```css
const throttle = (fn, time) => {
 let lastCalledAt = 0;

 return function(...args) {
  const context = this;
  const now = Date.now();
  const remainingTime = time - (now - lastCalledAt);

  if (remainingTime <= 0) {
   fn.call(context, ...args);
   lastCalledAt = now;
  }
 }
}

const handleKeyDown = async (e) => {
 const { value } = e.target;
 // save it DB and also notify other peers
 await save(value);
}

<Editor onKeyDown={throttle(handleKeyDown, 1000)} />
```


我们已经修改了现有的代码来使用节流函数。它接受两个参数：

- fn：要进行节流的实际函数。
- time：函数允许执行的时间间隔。

实现方法很简单。我们将上次调用函数的时间存储在lastCalledAt中。下次调用函数时，我们检查是否已经过了时间，只有在这种情况下我们才执行fn。


我们已经接近成功了，但是这个实现有一个错误。如果在时间间隔内进行了最后一次带有一些数据的函数调用，之后没有再调用。根据我们当前的实现，我们将丢失一些数据。


为了解决这个问题，我们将参数存储在另一个变量中，并在没有接收到事件时启动一个延迟调用的定时器。


```css
const throttle = (fn, time) => {
 let lastCalledAt = 0;
 let lastArgs = null;
 let timeoutId = null;

 return function(...args) {
  const context = this;
  const now = Date.now();
  const remainingTime = time - (now - lastCalledAt);

  if (remainingTime <= 0) {
   // call immediately
   fn.call(context, ...args);
   lastCalledAt = now;
   if (timeoutId) {
     clearTimeout(timeoutId);
     timeoutId = null;
   }
  } else {
    // call later if no event is received
    lastArgs = args;
    if (!timeoutId) {
      timeoutId = setTimeout(() => {
        fn.call(context, ...lastArgs);
        lastCalledAt = Date.now();
        lastArgs = null;
        timeoutId = null;
      }, remainingTime);
    }
  }
 }
}
```


## 总结

- 防抖和节流是性能优化技术。
- 这两种技术都基于相似的原则-延迟执行。
- 防抖在接收到最后一个事件后等待t时间，而节流会在t时间内周期性地执行函数。
- 防抖技术常用于搜索功能，而节流技术常用于实时应用程序（不仅限于此）。
